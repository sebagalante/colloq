Colloq — System Architecture

Interactive diagram. Hover nodes to highlight paths and dependencies.
Architecture: Phoenix monolith on single VPS (Hetzner CX22). Postgres 17 + pg_search (ParadeDB). Caddy reverse proxy. Turnilio sidecar for bot detection. Infisical Cloud for secrets. No Docker in production. All external APIs behind adapter modules with Cachex TTL layers.
Solid = direct call / dependency Dashed = broadcast / async / PubSub
Request Flow — Match Day Goal
1. ScoreBotWorker polls API-Football every 90s
GET /fixtures/events — goal detected, new event in response
2. Worker creates system post + broadcasts PubSub
{:match_event, topic_id, %{type: "Goal", player, minute}}
3. Phoenix PubSub pushes to subscribed LiveViews
Match Day LiveView receives event — triggers goal alert banner + SVG balloon shower
4. PushNotificationWorker fires Web Push (PWA)
Subscribed users get "⚽ GOOOL DE LA ACADEMIA!" off-screen notification
5. Postgres stores the event post permanently
Archive forever. Post is part of the match thread history.
Request Flow — LLM Bot @mention
1. User posts "@buscabot últimas noticias de Racing"
Tiptap stores mention as MarkView node with user_id metadata
2. MentionTriggerWorker detects @mention on create
Looks up bot_persona by slug, checks rate limit per user
3. LlmResponderWorker loads persona from DB
BuscaBot → provider: openrouter, model: llama-3.3-70b-instruct:free + web_search tool
4. Colloq.LLM calls OpenRouter API via Req
Model decides to search web (Exa), synthesizes answer, ~150-400 tokens
5. Bot reply posted as new post in thread
PubSub streams tokens if streaming enabled. Stored body_json + HTML.
Data TTL Strategy
Active season player stats — Cachex 6h TTL
SofascoreWorker refreshes on demand / comparison page visits
Finished season stats — Cachex 30-day TTL
Stats never change after season ends. Effectively permanent cache.
Fixture lists / standings — Cachex 12h TTL
ScoreBotWorker + SofascoreWorker warm cache daily
Live match events — no cache
Direct API-Football polling every 90s, broadcast immediately
XFeed dedup — Cachex 7-day TTL
Prevents duplicate ingestion of same tweet
Security Boundaries
Caddy terminates TLS and routes by path
:80/:443 → Phoenix :4000, Turnilio :4001. No SSL in application code.
Turnilio validates every POST / sensitive action
SHA-256 PoW + blake3 token signing. Between Caddy and Phoenix.
Admin routes gated by VpnOnly plug + :admin_network pipeline
ADMIN_ALLOWED_CIDRS restricts /admin/* to office / VPN IPs only
Infisical injects secrets — never touch disk
Only machine identity token stored on VPS. infisical run -- exec.
LLM API keys in site_settings (DB), not .env
Masked in UI. Rotatable without deploy. Test connection before save.
